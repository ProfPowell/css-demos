07 - Overflow

Overflow dictates how content is treated if its height or width exceeds that of its containing parent.

There are 4 overflow values you can set on a block element:

  • visible
  • hidden
  • scroll
  • auto

Visible (Default)

By default, block elements take the width of their parent. If no parent elements have width set, the block element's width is that of the device width.
Their height is different. Instead of taking the height of their parent, and thus ultimately the document's itself, it takes the height of its child content. As the inner content grows and shrinks, so does the height of the block element. Since the content is never restricted, the default overflow state of a block element is said to be visible. You may omit adding overflow: visible;, as this is the default. Only explicitly set it if you are overriding a pre-existing overflow state on an element (either as a more specific selector override, or on-demand w/ JavaScript).

Ellipsis

In conjunction with overflow: hidden;, you can use CSS to cutoff text with an ellipsis: ... There are 3 properties that achieve this: 

overflow: hidden; 
text-overflow: ellipsis; 
white-space: nowrap;
  • overflow: hidden; - We've seen this. This limits the area inside the element where content can be shown; excess will be cutoff.
  • text-overflow: ellipsis; - This tells the device how to render the visual indicator of cutoff content. Here, it will be ...
  • white-space: nowrap; - Finally, tell the device not to wrap text, or you would need to set a fixed height on the parent element.

Note: The downside to white-space: nowrap; is that the visible content will be limited to one line.
