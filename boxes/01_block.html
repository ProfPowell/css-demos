At their core, web pages are documents. You encode your content with specific rules (HTML tags) that tell devices how to interpret and render it. Like a normal document, you define a hierarchy structure (tree) of weighted headlines and content (paragraphs, lists, tables, etc.).

Block tags

The basic building blocks of a document are block elements. These tags create boxes, or blocks, which may contain grouped child content. They may or may not have semantic value, based on the type of tag used.

The most common block elements are <div> and <p>. While both act as containers, their semantics dictate their usage.

  • <div> tags are generic layout containers that provide no semantic value. You cannot infer the inner content-grouping's meaning from the tag itsef.
  • <p> tags, however, tell the device to expect paragraph content, which in turn may have inline-element semantics. With this tag you can infer semantics from the tag itself.

By itself (without additional CSS or JavaScript), multiple block elements will never occupy the same horizontal space. Thus, block elements always 'stack' on top of each other, separated by line breaks.

As a container, block elements take the width of their parent environment, or scope, by default. In the above example, that parent is the <body> tag, which as a block element itself has the full device width. This pattern continues up the document hierarchy until the terminus is reached; the <html> tag.

Devices typically add base styling to different tags, once their semantics are determined.

  • <div> tags lack margins, since they act as generic grouping boxes
  • <p> tags tend to have a top and bottom margin applied, since the device anticipates their usage with other paragraph and headline tags.

In the above example, there are 2 <div> boxes that don't have vertical spacing (margins) by default. If you change the tags to <p>, you will see margins appear.

With CSS, you can turn any element into a block element with the display property: display: block;

Note: Devices tend to add their own styling to specific tags. Because this can be inconsistent across different browsers, you shouldn't rely on this alone. If the styling is important, it should be applied with CSS.
