06 - z-index

As we saw in the Position step, when two or more elements are given position, and compete for the same 2D space, the browser doesn't know which one to give priority to (and thus show above the others). By default, it favors the DOM element found lowest in the document hierarchy.

You may find that changing the order of the elements in the DOM is not desirable, or feasible. Instead, the CSS property z-index can be used to raise or lower the element's stacking priority (stacking context). Think of the device screen as a 2D plane (x and y), where z-index changes the stacking order of elements (z).

If you are familiar with the idea of layers in Photoshop, z-index behaves the same way; if you raise the the priority of a stacked layer, it comes to the front of view. If you lower it, it recedes behind other layers with a higher priority.

Below, we have two absolutely-positioned elements inside a relative parent, both set to occupy the same position:

Box 1 - position:absolute; top:0; left:0;
Box 2 - position:absolute; top:0; left:0;

As noted, Box 2 is visible because it comes lower in the document hierarchy.

To fix, we could reorder the box elements, or we can set z-index on Box 1:

Box 1 - z-index:0; position:absolute; top:0; left:0;
Box 2 - position:absolute; top:0; left:0;

Note: since the value of 0 (zero) indicates the default state of the element's local stacking context, you typically see z-index values set starting with 1 and increasing from there.

Child elements inside z-indexed parents

By default, all z-index values compete for the same space:

Box 1 - position:absolute; top:0; left:0; z-index:3; Higher z-index child
Box 2 - z-index:1; position:absolute; top:0; left:0;

Despite Box 2 having a z-index, and Box 1 not, Box 1's child element has a higher z-index set, which is why it appears above Box 2, which in turn appears above Box 1.

This general competition is severed only if sibling elements both have a z-index set. Once a z-index is set on an element, its child elements get their own local scope, that resets to zero. Therefore, to fix the above example, Box 1 (1) needs a lower z-index set than Box 2 (2). Then, despite Box 1's child element having a higher z-index than Box 2 (3), it doesn't matter because they are not competing in the same scope:

Box 1 - z-index:1; position:absolute; top:0; left:0; z-index:3; Higher z-index child
Box 2 - z-index:2; position:absolute; top:0; left:0;

Now the sub-child element doesn't appear, even though it has a higher z-index.

Notes:

  • Due to z-index scoping, it can quickly become difficult to figure out what is going on. The element you think should be higher up the stack could be failing because its 10th parent element has its own z-index set and is lower than one of its siblings.
    It is highly recommended to avoid z-index unless absolutely necessary, and if all other solutions have been exhausted.
  • Negative values can be set for z-index, but you may find yourself putting your element behind the document background, etc. Negative values are discouraged.
