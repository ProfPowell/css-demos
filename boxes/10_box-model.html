10 - Box Model

The CSS Box Model dictates how all of the parts of a box add together to determine its width and height. If you want your box to be 100 pixels wide, how should it be interpreted if you add margin, padding, and borders?

Should the edge-to-edge of the box never exceed 100px wide, or should the content area (content box) be 100px, and any additional margin, padding, and borders would be added outside of it, thus increasing the width past 100px?

Historically, both of these ways to interpret a box were used in major browsers, which caused problems for developers who wanted pixel-perfect performance across browsers, or who ran into issues of elements/content breaking down to the next line in some browsers, because their width was too large for the available horizontal space.

Luckily, all 4 modern western browsers (Chrome, Firefox, Safari, Edge) use the same model by default.

The 2 models are the Standard and Alternative models

Standard Model

The standard model dictates that the content area is the set fixed-width value, and margin, padding, and borders are added in addition, so the total width may exceed the fixed-width value 

.wrap { width:400px; height:200px; background:lightgrey; }
.el { width:400px; height:100px; background:indianred; border:10px solid blue; }

Alternative Model

The alternative model dictates that the total box area is the set fixed-width value, where margin, padding, and borders are included, so the total width may not exceed the fixed-width value.

To convert a box to the alternative model, add box-sizing: border-box;

.wrap { width:400px; height:200px; background:lightgrey; }
.el { width:400px; height:100px; box-sizing:border-box; background:indianred; border:10px solid blue; }
